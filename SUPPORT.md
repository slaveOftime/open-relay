# Support

## Getting help

For bugs, feature ideas, and workflow questions, open a GitHub issue with:

- what you were trying to do
- the exact command you ran
- your platform
- the output or error you saw

The more reproducible the report, the easier it is to help.

## Before opening an issue

- Read `README.md`
- Check `ARCHITECTURE.md` or `ARCHITECTURE_PTY.md` for deeper implementation detail
- Search existing issues first to avoid duplicates

## Best fit for issues

- installation problems
- session lifecycle bugs
- notification or prompt-detection problems
- remote supervision and federation questions
- feature requests tied to a concrete workflow
